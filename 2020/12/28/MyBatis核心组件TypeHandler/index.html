问题

我们在使用 JDBC 开发应用程序时，其中一个比较繁琐的步骤就是处理 JDBC 类型和 Java 类型之间的转换。例如以下两种情况：

  • PreparedStatement 对象在为参数占位符设置值时，需要调用 PreparedStatement 接口中的 setXXX() 方法将 Java 类型转换为对应的 JDBC 类型并未参数占位符赋值。

  • 当执行完 SQL 语句拿到 ResultSet 对象后，需要调用 ResultSet 对象的 getXXX() 方法将获取字段值转换为对应 Java 类型。

总结

说到底，TypeHandler 实现类还是调用了 JDBC API，只不过抽象出了公共接口，创建一个统一的外观更方便用户使用。而 Mybatis 在启动的时候注册了一些常用的数据类型对应 Handler。
